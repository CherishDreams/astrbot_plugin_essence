# AstrBot 神人语句加精插件

自动识别群聊中的"神人语句"或"经典语句"，并将其设为精华消息。支持自动分析、手动加精和主动分析三种模式。

## 功能特性

### 自动加精模式
- 📝 自动累积群消息到指定数量
- 🤖 调用 LLM 智能分析识别神人语句
- ⭐ 自动调用 OneBot API 设置精华消息
- 🎯 支持群白名单过滤

### 手动加精模式
- 💬 管理员回复消息后发送"加精"指令
- 🔢 管理员发送 `/加精 <message_id>` 指定消息加精

### 主动分析模式
- 📊 管理员发送 `/分析加精 <数量>` 获取历史消息并分析加精
- 🔍 可指定分析的消息数量（如 `/分析加精 100` 分析最近100条消息）
- ⚡ 适合快速回顾群内精华内容

### 加精识别标准
LLM 会从以下维度识别值得加精的内容：

**🎯 经典入典类**：
- 立Flag：发出预言或承诺后被"回旋镖"打脸
- 抽象话：语法混乱但意思奇妙的句子
- 入典语句：会被群友反复引用的经典语录
- 回旋镖：之前的预言被现实打脸

**💡 内容质量类**：
- 神人见解：一针见血、深刻透彻的观点
- 犀利吐槽：精辟批评、妙语连珠
- 知识干货：有价值的技术分享
- 幽默金句：让人会心一笑的段子

**📝 群文化相关**：
- 群内梗：与群文化相关的经典发言
- 高能对话：值得反复回看的对话片段

## 安装

### 方法一：通过 AstrBot 插件市场安装
在 AstrBot 管理面板的插件市场中搜索"神人语句加精"并安装。

### 方法二：手动安装
1. 克隆或下载本仓库
2. 将 `astrbot_plugin_essence` 目录放入 AstrBot 的 `data/plugins/` 目录下
3. 在 AstrBot 管理面板启用插件

## 配置

在 AstrBot 管理面板的插件配置页面可以调整以下选项：

### 基础设置
| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 启用自动加精模式 | 自动累积消息并调用 LLM 分析 | true |
| 启用手动加精模式 | 管理员可通过指令手动加精 | true |
| 调试模式 | 输出详细日志便于排查问题 | false |
| 消息累积数量阈值 | 达到此数量触发 LLM 分析 | 50 |
| 每次分析最大加精数量 | 每次分析最多加精的消息数 | 3 |
| 群组白名单 | 限制只在指定群启用，留空则全部启用 | [] |

### 模型设置
| 配置项 | 说明 |
|--------|------|
| 分析模型提供商 | 用于判断神人语句的 LLM 模型，留空使用默认模型 |

### 提示词设置
| 配置项 | 说明 |
|--------|------|
| LLM 判断提示词模板 | 指导 LLM 识别神人语句的提示词 |

### 指令设置
| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 手动加精指令 | 回复消息后发送的指令 | 加精 |
| 按ID加精指令前缀 | 指定消息ID加精的指令 | /加精 |
| 主动分析最大消息数量 | `/分析加精` 指令最多获取的消息数 | 200 |

## 使用说明

### 自动加精
插件会自动监听群消息并累积，当达到配置的阈值后触发 LLM 分析。LLM 会从累积的消息中识别神人语句并自动加精。

**推荐配置**：
- 活跃群：设置阈值 30-50，避免频繁调用 LLM
- 闲聊群：设置阈值 50-100，减少无效分析

### 手动加精

**方式一：回复加精**
1. 管理员回复目标消息
2. 发送"加精"指令（可自定义）

**方式二：指定ID加精**
1. 发送 `/加精 <消息ID>`
2. 消息 ID 可通过 OneBot 实现获取（如 NapCat、LLOneBot）

**方式三：主动分析加精**
1. 发送 `/分析加精 <数量>`（如 `/分析加精 100`）
2. 插件会获取群历史消息，调用 LLM 分析并加精
3. 数量参数可选，默认使用配置的阈值，最大不超过 200 条

## 前置要求

- AstrBot 版本 >= 4.16
- 平台适配器：aiocqhttp (OneBot v11)
- OneBot 实现：NapCat、LLOneBot、go-cqhttp 等
- **机器人需要有设置精华消息的权限**（通常是群管理员或群主）

## 注意事项

1. **权限要求**：机器人必须是群管理员或群主才能设置精华消息
2. **模型选择**：建议选择响应快的小模型进行判断，减少延迟
3. **阈值设置**：阈值过低会导致频繁调用 LLM，阈值过高会导致错过神人语句
4. **白名单**：建议只在活跃且有价值的群启用，避免无效分析

## 技术细节

- 使用 AstrBot 插件 KV 存储缓存消息
- 通过 `AiocqhttpMessageEvent.bot.call_action()` 调用 OneBot API
- 支持 `Reply` 消息组件检测回复消息
- 使用 `asyncio.Lock` 防止并发分析冲突
